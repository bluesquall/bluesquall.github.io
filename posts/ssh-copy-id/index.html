Copying SSH keys to a remote machine

Ideally, both source and destination machines have ssh-copy-id available. In that case, just:

$ ssh-copy-id -i ~/.ssh/id_rsa user@host

If the destination machine does not have ssh-copy-id, you can pipe output from cat through ssh.

$ cat ~/.ssh/id_rsa.pub | ssh user@host "mkdir -p ~/.ssh; cat >> ~/.ssh/authorized_keys

If you already have an entry for the destination machine in your ~/.ssh/config and want to ignore the options specified there, use -o to override, or -F \dev\null to ignore the config entirely.
